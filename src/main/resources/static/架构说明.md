# 双页面架构说明

## 📋 概述

本系统采用**双页面架构**，分为用户端和管理端两个独立的HTML页面，共享同一套数据存储。

---

## 🎯 页面划分

### 1. 👥 用户端（index.html）

**定位**：面向普通用户的只读页面

**功能模块**：
- ✅ 查看公告（只读）
- ✅ 使用数据计算器（基础计算、统计分析、单位转换）
- ✅ 浏览开发者数据（只读、可搜索筛选）

**特点**：
- 简洁清爽的紫色渐变主题
- 无编辑和删除功能
- 可通过导航栏进入管理端

**访问方式**：
- 直接打开 `index.html`
- 或从管理端点击"👥 用户端"链接

---

### 2. 🔐 管理端（admin.html）

**定位**：面向管理员的管理后台

**功能模块**：
- ✅ 公告管理（发布、编辑、删除）
- ✅ 数据管理（添加、编辑、删除）
- ✅ 数据统计（实时显示数据概览）

**特点**：
- 醒目的红色渐变主题（区别于用户端）
- 完整的CRUD操作功能
- 数据统计面板
- 每条数据都有编辑和删除按钮

**访问方式**：
- 从用户端点击"🔐 管理者入口"按钮
- 或直接打开 `admin.html`

---

## 🔄 数据共享机制

### LocalStorage 数据存储

两个页面共享以下数据：

```javascript
// 公告数据
localStorage.getItem('announcements')

// 开发者数据
localStorage.getItem('developerData')
```

### 数据流向

```
管理端（写入） → LocalStorage → 用户端（读取）
     ↑                                    ↓
     └────────── 实时同步 ──────────────┘
```

**关键点**：
- ✅ 管理端发布的数据，用户端立即可见
- ✅ 用户端刷新页面即可获取最新数据
- ✅ 两个页面的数据完全同步
- ⚠️ 仅限同一浏览器内共享

## 🔧 技术实现

### 用户端（index.html）

**加载的脚本**：
```html
<script src="js/app.js"></script>
```

**核心功能**：
- `renderAnnouncements()` - 渲染公告（无删除按钮）
- `renderDeveloperData()` - 渲染数据（无删除按钮）
- 计算器功能（完整）
- 搜索和筛选功能

### 管理端（admin.html）

**加载的脚本**：
```html
<script src="js/app.js"></script>
<script src="js/admin.js"></script>
```

**核心功能**：
- `renderAnnouncements()` - 渲染公告（带编辑/删除按钮）**覆盖app.js中的同名函数**
- `renderDeveloperData()` - 渲染数据（带编辑/删除按钮）**覆盖app.js中的同名函数**
- `editAnnouncement()` - 编辑公告
- `editDeveloperData()` - 编辑数据
- `updateDataStats()` - 更新数据统计
- 所有用户端功能 + 管理功能

## 🚀 工作流程

### 典型使用场景

```
1. 管理员打开 admin.html
   ↓
2. 发布公告 / 添加数据
   ↓
3. 数据自动保存到 LocalStorage
   ↓
4. 用户打开 index.html
   ↓
5. 自动从 LocalStorage 读取并显示最新数据
```

### 实时更新

- **同浏览器**：管理端修改后，用户端刷新即可看到变化
- **不同浏览器/设备**：数据不共享（LocalStorage限制）

## 🔒 权限控制

### 当前实现
- ⚠️ **无登录验证**：任何人都可以访问管理端
- ⚠️ **无权限隔离**：用户可以直接打开 admin.html

### 建议改进（可选）

如需增加安全性，可以：

1. **简单密码保护**
   ```javascript
   // admin.html 加载时
   const password = prompt('请输入管理密码');
   if (password !== 'your-password') {
       window.location.href = 'index.html';
   }
   ```

2. **隐藏管理入口**
   - 移除用户端的"管理者入口"按钮
   - 仅通过直接输入 URL 访问

3. **使用后端验证**
   - 部署真正的用户认证系统
   - 使用 JWT 或 Session 管理权限

## 💡 最佳实践
