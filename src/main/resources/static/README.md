# 信息发布与数据管理平台

这是一个纯静态网页应用，包含**用户端**和**管理端**两个独立页面。

## 页面架构

### 👥 用户端（index.html）
普通用户可以访问的页面，功能包括：
- 📢 **查看公告**：浏览管理者发布的最新公告
- 🐾 **宠物收益率**：计算宠物在出租、发车、PVP三种模式下的收益率
- 💻 **查看开发者数据**：浏览管理者发布的数据（只读）

### 🔐 管理端（admin.html）
管理者专用的后台页面，功能包括：
- 📢 **公告管理**：发布、编辑、删除公告
- 💻 **数据管理**：添加、编辑、删除开发者数据
- 📊 **数据统计**：实时查看数据统计信息

## 功能特点

### 用户端功能

#### 1. 📢 查看公告
- 浏览最新公告列表
- 支持普通、重要、紧急三种优先级显示
- 按时间倒序排列

#### 2. 🐾 宠物收益率计算器
专业的宠物投资回报分析工具，支持三种计算模式：

##### 📊 市场数据展示
- 实时显示金杯最低价
- 实时显示宠物最低价
- 实时显示1金杯可兑换灵石数量
- 支持编辑和保存市场数据
- 数据自动保存在浏览器本地

##### 🏠 出租模式
- 计算公式：收益率 = 出租收益灵石 / 金杯兑换灵石 × 金杯单价 / 宠物价格
- 适合长期稳定收益的投资者

##### 🚗 发车模式
- 计算公式：收益率 = 每天发车灵石收益 / 金杯兑换灵石 × 金杯单价 / 宠物价格
- 适合日常活跃玩家

##### ⚔️ PVP模式
- 计算公式：收益率 = 掉落金杯数 × 金杯价格 / 宠物价格
- 适合喜欢战斗的玩家

##### 智能功能
- 自动计算回本天数
- 智能评级系统（优秀🌟/良好👍/一般⚠️）
- 详细展示计算公式
- 响应式设计，支持所有设备

#### 3. 💻 查看开发者数据
- 浏览管理者发布的数据
- 搜索和筛选功能
- 卡片式展示
- 支持文本、数字、链接三种类型

### 管理端功能

#### 1. 📢 公告管理
- 发布公告（支持普通、重要、紧急三种优先级）
- 编辑已发布的公告
- 删除公告
- 实时统计公告数量
- 数据自动保存在浏览器本地存储中

#### 2. 🐾 宠物收益率计算器
- 与用户端功能相同
- 方便管理员快速计算收益率
- 支持三种模式：出租、发车、PVP
- 智能评级和回本天数计算

#### 3. 💻 数据管理
- 添加开发者数据（支持文本、数字、链接三种类型）
- 编辑已发布的数据
- 删除数据
- 分类管理数据
- 实时数据统计（总数、分类数、各类型数量）
- 搜索和筛选功能
- 数据自动保存在浏览器本地存储中

## 使用方法

### 本地使用
1. 直接在浏览器中打开 `index.html` 文件
2. 或者使用本地服务器运行（推荐）

### 部署到网络（让其他人可以访问）

#### 方法一：使用 GitHub Pages（免费推荐）
1. 创建一个新的 GitHub 仓库
2. 将 `static` 文件夹中的所有文件上传到仓库
3. 进入仓库设置 → Pages → 选择主分支
4. 获得一个类似 `https://用户名.github.io/仓库名` 的网址
5. 将这个网址分享给其他人

#### 方法二：使用 Netlify（免费推荐）
1. 访问 https://www.netlify.com
2. 注册账号
3. 直接将 `static` 文件夹拖拽到 Netlify 页面
4. 获得一个可访问的网址
5. 可以自定义域名

#### 方法三：使用 Vercel（免费推荐）
1. 访问 https://vercel.com
2. 注册账号
3. 导入项目或拖拽文件夹
4. 获得一个可访问的网址

#### 方法四：使用本地服务器测试
```bash
# 如果安装了 Python
cd src/main/resources/static
python -m http.server 8080

# 或者使用 Node.js 的 http-server
npm install -g http-server
cd src/main/resources/static
http-server -p 8080
```

然后在浏览器访问：`http://localhost:8080`

✅ 数据保存在浏览器本地，刷新不丢失  
✅ 美观的渐变色界面  
✅ 流畅的动画效果  
✅ 完全免费部署  

## 注意事项
1. **数据共享**：用户端和管理端的数据是共享的，都存储在浏览器的 LocalStorage 中
2. **管理者权限**：只有管理端可以发布、编辑、删除公告和数据
3. **用户权限**：用户端只能查看公告和使用计算器，不能修改数据
4. **数据持久化**：数据保存在浏览器 LocalStorage，清除浏览器数据会丢失
5. **跨设备**：不同设备之间数据不共享
6. **建议定期备份**：重要数据建议定期备份
